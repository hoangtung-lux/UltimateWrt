# Device Guide: Intel N100 Mini PC (x86_64)

## Tổng quan

Intel N100 Mini PC là platform x86_64 cho soft router throughput cao, hợp 2.5GbE, multi-WAN, VPN và nhiều package. Track N100 hiện tại dùng OpenWrt 24.10.x/kernel 6.6.x.

## Flash Procedure

### Cảnh báo

`*.img.gz` là raw disk image. Ghi sai ổ sẽ mất dữ liệu/partition table. Chỉ flash vào USB/SSD test riêng.

### Windows

Dùng Rufus/balenaEtcher/Win32 Disk Imager, chọn file:

```text
openwrt-x86-64-generic-ext4-combined-efi.img.gz
```

Chọn đúng USB/SSD test. Nếu tool hỏi mode, chọn raw/DD mode.

### Linux

```bash
lsblk
# Ví dụ ổ test là /dev/sdX. Kiểm tra thật kỹ trước khi chạy.
gzip -dc openwrt-x86-64-generic-ext4-combined-efi.img.gz | sudo dd of=/dev/sdX bs=4M status=progress conv=fsync
sync
```

## Network Interface Mapping

Không hardcode interface. N100 có thể hiện `eth*` hoặc `enp*` tùy boot/cmdline/model.

Chạy:

```bash
ip link
lspci -nn | grep -i ethernet
ls -la /sys/class/net/*/device
for i in /sys/class/net/*; do echo "$(basename $i) -> $(readlink -f $i/device)"; done
```

Cắm dây từng cổng và xem link:

```bash
ethtool <IFNAME> | grep -i 'Link detected\|Speed'
```

Ghi report:

```text
IFNAME = cổng vật lý nào
Speed = 1000/2500Mb/s
Link detected = yes/no
```

## WAN/LAN template sau khi đã map interface

Ví dụ `WAN_IF=<WAN_IF>` và `LAN_IF=<LAN_IF>`:

```bash
# Bridge LAN
uci set network.lan_dev='device'
uci set network.lan_dev.name='br-lan'
uci set network.lan_dev.type='bridge'
uci add_list network.lan_dev.ports='<LAN_IF>'

uci set network.lan='interface'
uci set network.lan.device='br-lan'
uci set network.lan.proto='static'
uci set network.lan.ipaddr='192.168.1.1'
uci set network.lan.netmask='255.255.255.0'
```

PPPoE/VLAN theo ISP xem ISP-CONFIG.md.

## IRQ / performance

N100 nên bật `irqbalance`:

```bash
service irqbalance enable
service irqbalance start
cat /proc/interrupts | grep -Ei 'igc|r8125|eth|enp'
```

Packet steering:

```bash
uci set network.globals.packet_steering='1'
uci commit network
/etc/init.d/network restart
```
